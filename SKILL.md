# 判决书分析技能 v2

分析本地判决书文档，提取案件关键信息，生成结构化的摘要报告和综合分析报告。

## 工作流程

### 步骤 1：接收输入

用户指定判决书路径，支持：
- **单文件**：`/path/to/judgment.pdf`
- **文件夹**：`/path/to/cases/`（批量处理所有 PDF/Word 文件）
- 支持格式：PDF、Word (.docx/.doc)

### 步骤 2：批量处理

运行 `scripts/analyzer.py`：

```bash
python3 ~/.claude/skills/judgment-analyzer/scripts/analyzer.py <输入路径> [输出目录]
```

脚本自动：
1. 读取每个判决书文件
2. 提取关键信息（案号、法院、案由、当事人、判决结果等）
3. 生成结构化摘要

### 步骤 3：生成报告

每个判决书生成两个摘要文件（存放在单独的摘要文件夹）：
- `案件名_摘要.txt` - 文本格式摘要
- `案件名_摘要.docx` - Word 格式摘要

最后生成综合分析报告：
- `综合分析报告.docx` - 包含所有案件的对比分析

## 输出结构

```
输入文件夹/
├── 综合分析报告.docx          # 综合分析报告
├── 案件1_摘要.txt            # 单个案件摘要（文本）
├── 案件1_摘要.docx           # 单个案件摘要（Word）
├── 案件2_摘要.txt
├── 案件2_摘要.docx
└── ...
```

## 综合分析报告内容

**一、案件概览**
- 案由分布统计
- 法院分布统计
- 裁判日期范围

**二、案件对比分析**
- 共同特征归纳
- 差异分析（含对比表格）
- 法律要点归纳

**三、结论与建议**
- 案件特征总结
- 法律实践建议

## 使用示例

### 触发方式
- 「分析判决书 /path/to/case.pdf」
- 「批量分析 /Users/xxx/Desktop/cases/」
- 「生成判决书综合报告」
- 「对比分析这几份判决书」

### 完整流程

```
用户：批量分析 ~/Desktop/casetest/ 文件夹

助手：
1. 运行分析脚本
2. 脚本自动创建 ~/Desktop/casetest_摘要/ 文件夹
3. 为每份判决书生成摘要
4. 生成综合分析报告
5. 展示分析结果统计
```

## 信息提取内容

每个案件提取以下信息：
- **案号**：如（2023）鲁民终163号
- **法院**：审理法院名称
- **案由**：案件类型
- **裁判日期**：判决时间
- **当事人**：原告/被告或上诉人/被上诉人
- **判决结果**：判决主文
- **关键事实**：案件经过

## 注意事项

- 脚本路径：`~/.claude/skills/judgment-analyzer/scripts/analyzer.py`
- 首次使用需安装依赖：`pip install -r ~/.claude/skills/judgment-analyzer/scripts/requirements.txt`
- 批量处理时，自动跳过非判决书文件
- 摘要文件夹默认创建在输入文件夹同级目录
